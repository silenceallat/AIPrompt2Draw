## 🤔 基础问题

### Q1: AIPrompt2Draw是什么？

**A**: AIPrompt2Draw是一个零安装、秒开的AI流程图生成器。你只需要用自然语言描述想要创建的流程图、架构图或UML图，AI就会自动生成DrawIO格式的图表代码，并直接加载到内置的编辑器中。

### Q2: 这个应用完全免费吗？

**A**: AIPrompt2Draw应用本身是完全免费和开源的。但使用时需要调用AI服务商的API，这可能会产生费用。各服务商的收费标准和免费额度不同，建议选择适合自己需求的服务商。

### Q3: 我的数据安全吗？

**A**: 非常安全！我们采用以下安全措施：
- ✅ 所有配置和对话记录都存储在您的本地浏览器中
- ✅ 我们不收集、不存储任何用户数据
- ✅ 代码完全开源，安全可审计
- ✅ API密钥仅用于调用AI服务，不会被记录

### Q4: 需要安装什么软件吗？

**A**: 完全不需要！AIPrompt2Draw是纯前端应用，您只需要：
1. 一个现代的浏览器（Chrome、Firefox、Safari、Edge等）
2. 稳定的网络连接
3. 一个AI服务商的API密钥

### Q5: 支持哪些浏览器？

**A**: 推荐使用以下浏览器的最新版本：
- Chrome 80+
- Firefox 75+
- Safari 13+
- Edge 80+

不支持IE浏览器。

## ⚙️ API配置

### Q6: 如何获取API密钥？

**A**: 不同服务商的获取方式：

**硅基流动（推荐新手）**：
1. 访问 [siliconflow.cn](https://siliconflow.cn/)
2. 注册账号并完成实名认证
3. 在控制台 → API密钥页面创建新密钥

**OpenRouter**：
1. 访问 [openrouter.ai](https://openrouter.ai/)
2. 注册账号
3. 在设置页面找到API Keys

**Kimi**：
1. 访问 [kimi.moonshot.cn](https://kimi.moonshot.cn/)
2. 登录或注册账号
3. 在开放平台获取API密钥

### Q7: 推荐使用哪个AI服务商？

**A**: 根据不同需求推荐：

**新手用户**：
- 硅基流动：国产稳定，价格优惠，中文友好
- 智谱GLM：专门优化的中文大模型

**高级用户**：
- OpenRouter：模型选择丰富，支持多种AI服务
- Claude系列：理解能力强，生成质量高

**预算有限用户**：
- 硅基流动：性价比最高
- 选择成本较低的模型

### Q8: API费用如何计算？

**A**: 费用通常按以下方式计算：
- **输入Tokens**：你的提示词长度
- **输出Tokens**：AI生成的内容长度
- **计费方式**：每1000个Tokens的价格

**省钱技巧**：
1. 关闭"携带历史对话"
2. 使用简洁明确的提示词
3. 选择性价比高的模型

### Q9: API密钥会被盗用吗？

**A**: 安全性很高：
- 密钥只存储在你的本地浏览器中
- 不会上传到任何服务器
- 采用HTTPS加密传输
- 建议定期更换密钥
- 不要在公共场所的电脑上保存密钥

### Q10: 可以同时配置多个服务商吗？

**A**: 可以！应用支持多服务商配置：
1. 每个服务商的API密钥会分别保存
2. 可以随时切换使用不同的服务商
3. 支持导入导出所有配置

## 💡 使用技巧

### Q11: 如何写出好的提示词？

**A**: 好的提示词应该包含：

**基础结构**：
```
[图表类型] + [主要元素] + [连接关系] + [可选要求]
```

**示例**：
```
创建一个用户注册流程图：
1. 用户输入注册信息
2. 系统验证信息格式
3. 发送验证码到邮箱
4. 用户输入验证码
5. 注册成功，跳转到登录页
使用蓝色配色，矩形表示处理步骤
```

**技巧**：
- 具体描述每个步骤
- 说明判断条件和分支
- 指定期望的样式或布局
- 提供业务背景信息

### Q12: 支持哪些类型的图表？

**流程图**：

**架构图**：

**UML图**：

**其他**：

### Q13: 生成的图表如何编辑？

**A**: 多种编辑方式：

**在线编辑**：
- 直接在左侧DrawIO编辑器中编辑
- 支持拖拽、调整、修改样式
- 可添加更多元素和连接

**导出编辑**：
- 导出为XML格式
- 在diagrams.net桌面版中编辑
- 导出为其他格式继续编辑

**手动调整**：
- 复制XML代码
- 在其他绘图工具中重新导入

### Q14: 如何保存图表？

**A**: 多种保存方式：

**直接保存**：
- 在DrawIO编辑器中：文件 → 保存
- 可保存到本地或云存储

**导出为图片**：
- 文件 → 导出 → PNG/JPEG/SVG
- 适合用于文档和演示

**导出为代码**：
- 文件 → 导出 → XML
- 保留完整结构，便于后续编辑

### Q15: 历史对话有什么用？

**A**: 历史对话功能：

**开启时**：
- AI会记住之前的对话内容
- 支持多轮修改和完善
- 生成更连贯的图表
- 消耗更多Tokens

**关闭时**：
- 每次对话都是独立的
- 节省API费用
- 适合简单的需求

**使用建议**：
- 复杂需求时开启
- 简单需求时关闭
- 根据预算灵活调整

## 🔧 技术问题

### Q16: 为什么DrawIO编辑器显示空白？

**A**: 可能的原因和解决方案：

**可能原因**：
1. DrawIO还在加载中
2. 网络连接问题
3. 浏览器缓存问题
4. 防火墙阻止访问

**解决方法**：
1. 等待10-15秒让编辑器完全加载
2. 刷新页面重试
3. 清理浏览器缓存
4. 检查网络连接
5. 尝试使用"复制XML"功能

### Q17: API调用失败怎么办？

**A**: 排查步骤：

**1. 检查配置**：
- API密钥是否正确
- 服务商选择是否正确
- 模型是否支持

**2. 检查网络**：
- 网络连接是否正常
- 是否有代理或VPN影响
- 服务商服务是否正常

**3. 检查余额**：
- API账户是否有足够余额
- 是否达到调用限制

**4. 尝试其他方案**：
- 切换到其他服务商
- 使用备用API密钥
- 简化提示词重试

### Q18: 生成的XML格式有问题？

**A**: 可能原因：

**AI生成问题**：
- 提示词不够明确
- 逻辑过于复杂
- AI模型理解偏差

**解决方法**：
1. 更详细地描述需求
2. 分步骤描述复杂逻辑
3. 提供具体的格式要求
4. 重新生成尝试
5. 手动修正XML代码

### Q19: 在移动设备上使用有问题？

**A**: 移动端使用建议：

**推荐方式**：
- 使用横屏模式
- 确保良好的网络连接
- 使用最新版移动浏览器

**限制**：
- 小屏幕编辑可能不便
- 复杂操作可能困难
- 建议简单操作使用

**替代方案**：
- 在移动设备上生成
- 导出后在电脑上编辑

### Q20: 如何备份数据？

**A**: 备份建议：

**配置备份**：
- 导出API配置到文件
- 保存到安全位置
- 多设备同步时使用

**图表备份**：

## 🔧 故障排除

### Q21: 页面无法加载？

**1. 检查网址**：
- 确认网址输入正确

**2. 检查浏览器**：

**3. 检查网络**：

- 尝试其他网络

### Q22: 发送消息没有响应？

**A**: 可能原因：

**API问题**：
- API密钥未配置或错误
- 服务商服务异常
- 网络连接问题

**解决方法**：
1. 检查API配置
2. 测试网络连接
3. 尝试其他服务商
4. 查看浏览器控制台错误信息

### Q23: 图表显示不完整？

**A**: 解决方案：

**DrawIO编辑器**：
- 等待完全加载
- 调整缩放比例
- 检查XML完整性

**内容问题**：
- AI生成可能不完整
- 重新生成更详细的图表
- 手动添加缺失元素

### Q24: 消耗的Tokens过多？

**A**: 优化建议：

**提示词优化**：
- 使用简洁明确的描述
- 避免冗余信息
- 分步骤描述复杂需求

**配置优化**：
- 关闭历史对话
- 选择经济型模型
- 避免频繁重新生成

**使用策略**：
- 先简单后复杂
- 逐步完善图表
- 复用成功的提示词
